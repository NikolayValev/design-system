# Command Panel Phase 3 — Reference Host (`apps/command-panel`) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the reference host `apps/command-panel` — a Vite + React SPA mounting `<CommandPanel>` plus two Vercel-function API routes wrapping the engine, wired to a read-only `DataRegistry` of the monorepo's own metrics, deploy-ready and verified by a propose→pin→reload e2e smoke.

**Architecture:** A new workspace app (`@apps/command-panel`). The client is a Vite-built React page mounting the engine's `<CommandPanel>` (from `@nikolayvalev/command-panel`). Two web-standard handlers — chat (`createCommandPanelHandler`) and data (`createDataRouteHandler`) — are exposed as Vercel functions under `api/*.ts` and, in local dev, via a Vite dev-middleware adapter (Node `req/res` ↔ web `Request/Response`). The `DataRegistry` exposes monorepo metrics: design-system component/section/page counts, the 12 visions across 5 families, Storybook story count, and design-system bundle sizes. Filesystem-derived metrics are captured at build time into a committed-and-regenerated `metrics.generated.json` (deploy-safe; real data, refreshed per build); visions are pure in-package data from the design system. This matches the repo's existing Vercel pattern (mcp-server `api/*.ts` + `vercel.json`; storybook static build) — the repo has no Next.js.

**Tech Stack:** Vite 5 + `@vitejs/plugin-react`, React 18, `@nikolayvalev/command-panel` (engine, `workspace:*`), `@nikolayvalev/design-system` (`workspace:*`), Vitest + jsdom + `@testing-library/react`, `fast-glob` + `tsx` (build-time metrics), Vercel functions (web signature). LLM via the engine: `claude-opus-4-8` (`ANTHROPIC_API_KEY` at runtime).

## Global Constraints

- **App identity:** package name `@apps/command-panel`, `private: true`, `type: "module"`. Workspace already globs `apps/*`. tsconfig extends `@repo/config/tsconfig.react.json`; eslint is `@repo/config/eslint.config.cjs` (the base app config, same as `apps/storybook`).
- **No Next.js.** Use Vite for the SPA and Vercel **web-standard** function handlers (`export function POST(request: Request): Promise<Response>`), which map 1:1 onto the engine's `(Request) => Promise<Response>` handlers and stream cleanly. Mirror the mcp-server's `api/*.ts` + `vercel.json` layout; `maxDuration` is set in `vercel.json` (not in-file), as mcp-server does.
- **Engine boundary:** the client imports only from `@nikolayvalev/command-panel` (client entry). Only `api/chat.ts` and the dev middleware may import from `@nikolayvalev/command-panel/server`. The `ANTHROPIC_API_KEY` lives only in the server/runtime env, never in client code or the Vite bundle.
- **Read-only data, never fabricated.** Every `DataSource.load` returns real monorepo data (from `metrics.generated.json` or in-package `visionThemes`). The data route resolves only registered ids (the engine enforces this).
- **Metrics freshness:** `metrics.generated.json` is committed (so imports/typecheck always resolve) and regenerated by `pnpm metrics` (run from `prebuild` and `pretest`, and on Vite dev start). Tests assert shape + plausibility (counts > 0), not exact committed numbers.
- **Charts/data shape:** data sources that feed charts return `{ label: string; value: number }[]` so generated widgets can pass them straight to `BarChart`/`Donut`/etc.
- **Testing:** Vitest; DOM/integration tests start with `// @vitest-environment jsdom`; `globals: true` so `@testing-library/react`'s `afterEach(cleanup)` runs. No test may hit a live LLM or network — the chat path is tested with a `MockLanguageModelV4` (engine) and the e2e uses a canned-SSE transport (the verified Phase-2c fixture).
- **Deploy-ready:** ship `vercel.json` (static SPA + `api/**` functions) and `.env.example` (`ANTHROPIC_API_KEY`); do NOT deploy or add secrets here (that is a separate human-triggered step). Add the app to `.github/dependent-apps.json`.

```ts
import fg from 'fast-glob';
import { statSync, writeFileSync } from 'node:fs';
import path from 'node:path';
import { fileURLToPath } from 'node:url';

export interface Metrics {
  generatedAt: string;
  components: number;
  sections: number;
  pages: number;
  stories: number;
  bundle: { file: string; bytes: number }[];
}

const DS = 'packages/design-system';

async function countTsx(repoRoot: string, dir: string): Promise<number> {
  const files = await fg([`${dir}/**/*.tsx`], {
    cwd: repoRoot,
    ignore: ['**/*.stories.tsx', '**/*.test.tsx', '**/*.dom.test.tsx'],
  });
  return files.length;
}

export async function generateMetrics(repoRoot: string): Promise<Metrics> {
  const [components, sections, pages] = await Promise.all([
    countTsx(repoRoot, `${DS}/src/components`),
    countTsx(repoRoot, `${DS}/src/sections`),
    countTsx(repoRoot, `${DS}/src/pages`),
  ]);

  const storyFiles = await fg(['packages/**/*.stories.{ts,tsx}', 'apps/**/*.stories.{ts,tsx}'], {
    cwd: repoRoot,
    ignore: ['**/node_modules/**', '**/dist/**'],
  });

  const bundleFiles = await fg([`${DS}/dist/**/*.js`], { cwd: repoRoot, ignore: ['**/cli/**'] });
  const bundle = bundleFiles
    .map((file) => ({ file: path.basename(file), bytes: statSync(path.join(repoRoot, file)).size }))
    .sort((a, b) => b.bytes - a.bytes);

  return {
    generatedAt: new Date().toISOString(),
    components,
    sections,
    pages,
    stories: storyFiles.length,
    bundle,
  };
}

async function main(): Promise<void> {
  const here = path.dirname(fileURLToPath(import.meta.url));
  const repoRoot = path.resolve(here, '../../../..');
  const metrics = await generateMetrics(repoRoot);
  const out = path.join(here, 'metrics.generated.json');
  writeFileSync(out, `${JSON.stringify(metrics, null, 2)}\n`, 'utf8');
  // eslint-disable-next-line no-console
  console.log(`metrics written: ${out}`);
}

// Run as a script (tsx src/data/generate-metrics.ts), not when imported.
const invokedDirectly = process.argv[1] && fileURLToPath(import.meta.url) === path.resolve(process.argv[1]);
if (invokedDirectly) {
  void main();
}
```

    load: async () => ({ count: metrics.components }),
  },
  {
    id: 'sections.count',
    description: 'Number of design-system sections.',
    resultSchema: { count: 'number' },
    load: async () => ({ count: metrics.sections }),
  },
  {
    id: 'pages.count',
    description: 'Number of design-system pages.',
    resultSchema: { count: 'number' },
    load: async () => ({ count: metrics.pages }),
  },
  {
    id: 'stories.count',
    description: 'Number of Storybook stories across the monorepo.',
    resultSchema: { count: 'number' },
    load: async () => ({ count: metrics.stories }),
  },
  {
    id: 'visions.list',
    description: 'All vision themes: id, display name, and family.',
    resultSchema: { type: 'array', items: { id: 'string', name: 'string', family: 'string' } },

```ts
import type { IncomingMessage, ServerResponse } from 'node:http';
import type { Plugin } from 'vite';
import { POST as chatPOST } from '../../api/chat';
import { POST as dataPOST } from '../../api/data';

export async function toWebRequest(req: IncomingMessage): Promise<Request> {
  const method = req.method ?? 'GET';
  const url = `http://localhost${req.url ?? '/'}`;
  const hasBody = method !== 'GET' && method !== 'HEAD';
  let body: Buffer | undefined;
  if (hasBody) {
    const chunks: Buffer[] = [];
    for await (const chunk of req) chunks.push(chunk as Buffer);
    body = chunks.length ? Buffer.concat(chunks) : undefined;
  }
  return new Request(url, {
    method,
    headers: req.headers as Record<string, string>,
    body,
  });
}

export async function sendWebResponse(res: ServerResponse, web: Response): Promise<void> {
  res.statusCode = web.status;
  web.headers.forEach((value, key) => res.setHeader(key, value));
  if (!web.body) {
    res.end();
    return;
  }
  const reader = web.body.getReader();
  for (;;) {
    const { done, value } = await reader.read();
    if (done) break;
    res.write(Buffer.from(value));
  }
  res.end();
}

function mount(
  res: ServerResponse,
  handler: (req: Request) => Promise<Response>,
  webReq: Request,
  next: (err?: unknown) => void,
): void {
  handler(webReq)
    .then((webRes) => sendWebResponse(res, webRes))
    .catch((err) => next(err));
}

export function commandPanelDevApi(): Plugin {
  return {
    name: 'command-panel-dev-api',
    configureServer(server) {
      server.middlewares.use('/api/chat', (req, res, next) => {
        if (req.method !== 'POST') return next();
        toWebRequest(req as IncomingMessage)
          .then((webReq) => mount(res as ServerResponse, chatPOST, webReq, next))
          .catch(next);
      });
      server.middlewares.use('/api/data', (req, res, next) => {
        if (req.method !== 'POST') return next();
        toWebRequest(req as IncomingMessage)
          .then((webReq) => mount(res as ServerResponse, dataPOST, webReq, next))
          .catch(next);
      });
    },
  };
}
```
